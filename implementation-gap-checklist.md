# DRP 已实现/未实现核对清单（v2需求对齐）

## 已实现（可复用）

- 前端基础框架与页面骨架：系统配置、JIRA 数据获取、团队配置、历史项目、预测参数、预测结果。
- 预测结果展示：周趋势、累计、backlog、团队拆分、Excel 预览。
- 字段映射管理：本地保存、导入/导出 JSON。
- 历史项目缓存：本地缓存与基础趋势对比。
- Excel 导出服务：FastAPI + openpyxl 模板导出。

## 未实现/部分实现

- 真实 Jira 拉取链路：当前以 mock 数据为主。
- 真实预测服务：当前以 mock 预测为主。
- 预测版本持久化：页面按钮存在但未落库。
- 历史/预测/JIRA 实际统一对比：尚未形成完整闭环。
- 团队导入导出：按钮存在但未接线。
- 参数页自动识别相似项目：按钮存在但未接线。
- 历史页筛选/导出：按钮未接线。
- 交付工程化：安装包、升级迁移、备份清理机制未落地。

## P0 范围确认（本轮必须完成）

1. 前端从 mock 切到真实后端接口调用（Jira、预测、历史项目读取）。
2. 后端补齐 Jira 同步、预测计算、项目/周数据查询接口。
3. 建立预测版本数据模型并提供保存/查询/软删除接口。
4. 打通历史/预测/JIRA 实际统一对比与偏差分析（至少完成单项目对比主流程）。
5. 最小工程化能力：本地数据目录、数据库迁移入口、备份与恢复脚本。

## 非 P0（后续迭代）

- 真实 Jira 认证与全字段映射驱动的生产级采集。
- 复杂规则预警（多阈值、多阶段、多角色）。
- 安装包 UI 与自动升级体验。
